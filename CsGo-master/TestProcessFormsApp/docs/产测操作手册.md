# TestProcessFormsApp 产测操作手册（测试工程师版）

## 1. 适用对象
产线测试/调试人员，不要求阅读代码。

## 2. 测试前准备
1. 打开软件，确认型号、CAN 配置、网络正常。
2. 确认打印机在线（若离线可先测试，后续补打）。
3. 确认服务器可访问（若不可访问可先测试，后续补传）。

## 3. 正常测试步骤
1. 选择产品型号。
2. 点击开始测试。
3. 观察测试项逐项执行结果。
4. 测试完成后关注：
   - 是否上传成功
   - 是否标签打印成功

## 4. 新增 2F 测试项如何配置
在 CSV 增加一行：
- 测试方法：`写MEC`
- 被测数据：`FD 03 03`
- 判断数据1：`FE`（这里可改，代表 XX）
- 判断数据2：`A5`（可选）
- 判断数据3：`00`（可选）

说明：最终发送 `2F FD 03 03 A5 XX 00`。

## 5. 上传失败处理（补传）
现象：提示无法连接远程服务器。  
处理：
1. 等网络恢复。
2. 从菜单点击“补传追溯数据”。
3. 观察补传结果提示。

## 6. 打印失败处理（补打）
现象：打印机断开、标签打歪、打印失败。  
处理：
1. 打开“补打标签”窗口。
2. 可选操作：
   - 刷新列表
   - 补打选中
   - 全部补打
   - 按关键字补打一条（条码/追溯码/序列号/ECUID）
3. 确认补打成功后再放行。

## 7. 常见异常与处理
### 7.1 UDS超时
优先检查：
- 线束/供电
- CAN ID 配置
- 设备是否已进入会话模式

### 7.2 前几项无结果或顺序异常
- 检查测试项 CSV 的“测试顺序”是否连续。
- 重新加载型号配置再启动。

### 7.3 软件报 0x8007000B
- 这是架构不匹配问题，联系开发确认 x86 环境和 DLL。
